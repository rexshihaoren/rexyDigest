*   **Srishti Gautam, Arjun Radhakrishna, Sumit Gulwani** — SkillAxe: Sharpening LLM-Authored Agent Skills Through Evaluation-Guided Self-Refinement (Paper) — 2026-06-09 — [https://arxiv.org/abs/2606.10546](https://arxiv.org/abs/2606.10546)
    *   **TL;DR:** SkillAxe enables LLMs to iteratively refine their own agent skill documents, closing up to 67% of the gap to human-authored skills without requiring labels or rewards.
    *   **Takeaways:** SkillAxe decomposes skill quality into four interpretable dimensions: quality impact, trigger precision, instruction compliance, and solution-path coverage. It produces structured improvement briefs using only execution traces, without ground-truth labels or test suites. On SkillsBench, SkillAxe improves pass rates by 28% relative over unimproved LLM skills. In the wild on SpreadsheetBench, a SkillAxe-built skill library raised pass rate from 16.0% to 52.0% using only 22 skills. The method functions as a continuous improvement engine that learns from past agent trajectories.
    *   **Implication for Rex Ren:** This demonstrates that autonomous agents can bootstrap their own capabilities through structured self-reflection, marking a step toward self-improving AI systems.
    *   **CompositeScore (4.9) | Topics: Agent**

    *   **TL;DR:** ABC-Bench evaluates LLM agents on biosecurity tasks like robot scripting, DNA design, and screening evasion, showing they outperform human experts.
    *   **Takeaways:** LLM agents can automate wet-lab biology tasks with high proficiency. Agents excel at knowledge-based tasks but struggle with novel reasoning. The benchmark includes dual-use tasks highlighting biosecurity concerns. Wet-lab validation confirmed agents produced functional DNA assembly scripts.
    *   **Implication for Rex Ren:** Practitioners must consider that agentic AI is now capable of manipulating biological reality, demanding new frameworks for oversight and alignment.
    *   **CompositeScore (4.8) | Topics: Agent**

*   **Aman Sharma, Sushrut Thorat, Paras Chopra** — Frontier Coding Agents Use Metaprogramming to Adapt to Unfamiliar Programming Languages (Paper) — 2026-06-09 — [https://arxiv.org/abs/2606.10933](https://arxiv.org/abs/2606.10933)
    *   **TL;DR:** Frontier coding agents adapt to unfamiliar programming languages by using metaprogramming (writing Python to generate target code), with distilled strategies improving weaker agents; performance scales with interpreter calls and output tokens for strong agents.
    *   **Takeaways:** Strong coding agents like Claude Opus 4.6 and GPT-5.4 xhigh spontaneously use metaprogramming to solve tasks in esoteric languages. Forbidding metaprogramming causes large performance drops, indicating it is a critical adaptation strategy. Distilled helper code from strong agents can significantly boost weaker agents (Sonnet 4.6, GPT-5.4 mini) without exposing them to solved benchmarks. Extra interpreter calls and output tokens amplify useful strategies but do not create them from scratch in weak agents. The ability to build a working model of an unfamiliar language using tools and feedback highlights a path toward more general autonomous problem-solving.
    *   **Implication for Rex Ren:** This shows that agents can learn the 'rules of a new world' (like a simulation) by building meta-tools, offering a blueprint for creating systems that adapt to and manipulate arbitrary computational environments.
    *   **CompositeScore (4.8) | Topics: Agent**

Results highlight the need for agents that maintain context coherence and avoid cascading errors over extended tasks. Workflow-GYM provides a rigorous platform to measure progress in real-world economic agent deployment.

(Paper) — 2026-06-09 — [https://arxiv.org/abs/2606.10956](https://arxiv.org/abs/2606.10956)
    *   **TL;DR:** This paper introduces a benchmark based on China's NCRE office tasks and finds that even the best agentic LLM system scores only 68.8%, far below the human reference of 95.5%, exposing large gaps in document automation.
    *   **Takeaways:** 7 frontier LLMs score at most 36.6% in single-turn mode on standardized office tasks. An agentic system with execution feedback and iterative repair reaches 68.8%, but still lags behind human performance. Office automation demands long-horizon planning, precise formatting, and cross-app integration, which current AI finds difficult. The results indicate that code-generating LLMs are not yet reliable for professional-grade document manipulation.
    *   **Implication for Rex Ren:** Just as mastering physical reality requires precision, mastering digital document reality requires an agent’s reliability at fine-grained code control—this benchmark serves as a stress test for how close we are to AI that can manipulate the fabric of our computational world.
    *   **CompositeScore (4.7) | Topics: Agent**

*   **Yuchen Ling, Shengcheng Yu, Zhenyu Chen, Chunrong Fang** — Toward Secure LLM Agents: Threat Surfaces, Attacks, Defenses, and Evaluation (Paper) — 2026-06-09 — [https://arxiv.org/abs/2606.10749](https://arxiv.org/abs/2606.10749)
    *   **TL;DR:** This paper systematically reviews the security landscape for LLM agents, identifying key threats like prompt injection, control-flow hijacking, state corruption, and outlining defensive principles and evaluation gaps.
    *   **Takeaways:** LLM agents face critical security threats from prompt injection and tool-mediated control hijacking, with emerging risks in persistent state corruption and multi-agent propagation. Current defenses are useful but weakly compositional, failing to provide end-to-end security in complex, stateful deployments. Evaluation benchmarks underrepresent long-horizon, deployment-sensitive risks, limiting real-world security assessments. Secure agent design requires explicit trust boundaries, principled privilege control, and provenance-aware state management.
    *   **Implication for Rex Ren:** For those engineering reality-code manipulators, unsecured agents are a direct vulnerability—understanding these risks is essential to prevent adversarial corruption of the systems we build to steer computational worlds.
    *   **CompositeScore (4.7) | Topics: Agent**

*   **Andrew Kang, Priya Narasimhan** — Monte Carlo Pass Search: Using Trajectory Generation for 3D Counterfactual Pass Evaluation in Football (Paper) — 2026-06-09 — [https://arxiv.org/abs/2606.11120](https://arxiv.org/abs/2606.11120)
    *   **TL;DR:** Monte Carlo Pass Search (MCPS) evaluates football passes by generating counterfactual trajectories with a learned world model and value function to estimate possession value gain.
    *   **Takeaways:** Adapts a discrete-token autoregressive trajectory generator (SMART) from autonomous driving to multi-agent sports simulation. Demonstrates strong best-of-20 forecasting accuracy compared to baselines using limited public data. Introduces distribution-aware execution-surplus scores (mean-based, percentile-based) for pass attribution. Supports fully hypothetical rollouts, enabling downstream counterfactual evaluation tasks.
    *   **Implication for Rex Ren:** Practitioner-philosophers should note the use of simulation-based counterfactual evaluation in multi-agent settings, reflecting techniques for agent planning under uncertainty.
